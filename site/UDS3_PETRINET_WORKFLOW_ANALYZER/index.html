    Was ist Soundness?¶

    Ein Workflow-Net ist sound, wenn es folgende Bedingungen erf√ºllt:

    1. Option to complete
      Von jedem erreichbaren Zustand kann der End-Zustand erreicht werden
      ‚Üí Keine Deadlocks!

    2. Proper completion
      Wenn End erreicht, ist genau 1 Token in End-Place
      ‚Üí Sauberes Terminieren!

    3. No dead transitions
      Jede Transition ist auf mind. 1 Pfad von Start zu End erreichbar
      ‚Üí Keine toten Code-Pfade!

    Soundness-Stufen¶

    class SoundnessLevel(Enum):
    NOT_SOUND = "not_sound"          # ‚ùå Verletzt Bedingungen
    RELAXED_SOUND = "relaxed_sound"  # üü° Schwache Soundness
    SOUND = "sound"                   # ‚úÖ Klassische Soundness
    STRICT_SOUND = "strict_sound"     # ‚úÖ‚úÖ Strenge Soundness

    üìä Strukturelle Eigenschaften¶

    Der Analyzer erkennt folgende strukturelle Properties:

    1. Workflow-Net¶

    StructuralProperty.WORKFLOW_NET
    • Genau 1 Source-Place (ohne Preset)
    • Genau 1 Sink-Place (ohne Postset)
    • Alle Knoten auf Pfad Source ‚Üí Sink

    2. Free-Choice¶

    StructuralProperty.FREE_CHOICE
    • Wenn zwei Transitions gleichen Input-Place teilen,
      m√ºssen sie identischen Input haben
    • ‚Üí Keine "versteckten" Entscheidungen

    3. State Machine¶

    StructuralProperty.STATE_MACHINE
    • Jede Transition hat genau 1 Input und genau 1 Output Place
    • ‚Üí Sequentieller Ablauf ohne Parallelit√§t

    Soundness-Theorem (van der Aalst)¶

    Ein Workflow-Net (N, i, o) ist sound ‚ü∫
Das erweiterte Netz (N', i', o') ist live und bounded

Wobei N' = N ‚à™ {t*} mit:
- t* = neue Transition
- ‚Ä¢t* = {o} (Input: Sink)
- t*‚Ä¢ = {i} (Output: Source)
